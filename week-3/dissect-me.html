Version Control, Git, and GitHub

Snapshots in Time

1/2/2016

Version control is the concept of keeping track of the iterations of some product. In fact, you may be familiar with the basic concept already, either through the 5th edition of some textbook or perhaps the common editing process of a paper, naming it "paperv2.doc" or "paper(1).txt". In the concept of software, version control is the process of keeping track of the development of a program by maintaining a history of its versions. In response to the need for good version control, version control systems arose, such as Git, Mercurial, and SVN, which were focused on streamlining the process of keeping 'snapshots' of software, as well as adding features such as branching, commit messages, and efficient storage of snapshots.

One popular version control software proudct is Git. Git is a program that resides on your host computer, helping manage the record-keeping of your software's development. Git tracks the changes you've made by monitoring a specific folder containing your project, called a repository. Whenever changes are made inside the repository, you can ask git to bundle those changes into a commit, or a current snapshot of the software. If you'd like to develop changes without immediately applying them to your master copy, git allows you to make a series of commits that are independent of the 'master' work, called branching. These branches can be merged back into the master copy of the software once you are satisfied with the changes on the separate branch.

After you've made your commits, it is a good idea to store your repository somewhere outside of your host computer, usually online. While this is a good form of backing up data, this process also allows you to easily collaborate with individuals remotely from having a centralized online repo, while also displaying your code for the world to see and learn. GitHub is one such online repository storage system. Including the aforementioned benefits above, GitHub also allows functions useful for software development teams, such as bug reports, a wiki system, and the ability to request the merging of branches to the master code, called pull requests. By pushing your code from your local repository to GitHub's online repository, your project is readily backed up step-by-step, while also being augmented by tools meant to assist coding teams.
